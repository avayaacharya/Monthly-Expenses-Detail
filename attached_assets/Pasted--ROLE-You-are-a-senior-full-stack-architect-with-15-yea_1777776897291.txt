[CONTEXT]
A mid-size IT services company — "Acharya Technology" — wants to build a 
personal Monthly Expenses Tracker web application for its employees. 
The application helps employees log monthly income sources and expenses 
category-by-category, auto-calculates the monthly balance 
(income minus total expenses), and provides rule-based investment suggestions 
based on the remaining balance.

The application has the following zones and pages — all fully linked 
in a continuous user journey from landing to reporting:

  ZONE 1  →  Landing Page          (public, no login)
      ↓

      ↓  ↓  ↓
      ↓  ↓  └─── Menu: Investment Suggestion Page
      ↓  └─────── Menu: Report Page
      └────────── Income & Expense Entry Page

Every arrow above is a real, wired navigation — zero dead buttons.

SECTION B — SPRING BOOT BACKEND (backend/)

DATABASE SCHEMA:

  Table: users
    id            BIGSERIAL PRIMARY KEY
    employee_id   VARCHAR(20) UNIQUE NOT NULL
    email         VARCHAR(100) UNIQUE NOT NULL
    full_name     VARCHAR(100) NOT NULL
    password_hash VARCHAR(255) NOT NULL
    role          VARCHAR(20) DEFAULT 'EMPLOYEE'
    created_at    TIMESTAMP DEFAULT NOW()

  Table: income_types
    id            BIGSERIAL PRIMARY KEY
    name          VARCHAR(100) NOT NULL
    is_system     BOOLEAN DEFAULT TRUE
    user_id       BIGINT REFERENCES users(id)

  Table: expense_categories
    id            BIGSERIAL PRIMARY KEY
    name          VARCHAR(100) NOT NULL
    is_system     BOOLEAN DEFAULT TRUE
    user_id       BIGINT REFERENCES users(id)

  Table: monthly_entries
    id            BIGSERIAL PRIMARY KEY
    user_id       BIGINT REFERENCES users(id) NOT NULL
    month         INTEGER NOT NULL
    year          INTEGER NOT NULL
    total_income  DECIMAL(12,2)
    total_expense DECIMAL(12,2)
    balance       DECIMAL(12,2)
    savings_rate  DECIMAL(5,2)
    suggestion    TEXT
    created_at    TIMESTAMP DEFAULT NOW()
    UNIQUE (user_id, month, year)

  Table: income_items
    id                BIGSERIAL PRIMARY KEY
    entry_id          BIGINT REFERENCES monthly_entries(id) NOT NULL
    income_type_id    BIGINT REFERENCES income_types(id)
    income_type_name  VARCHAR(100)
    amount            DECIMAL(12,2) NOT NULL

  Table: expense_items
    id                   BIGSERIAL PRIMARY KEY
    entry_id             BIGINT REFERENCES monthly_entries(id) NOT NULL
    expense_category_id  BIGINT REFERENCES expense_categories(id)
    category_name        VARCHAR(100)
    description          VARCHAR(255)
    amount               DECIMAL(12,2) NOT NULL
